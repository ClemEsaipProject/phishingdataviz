PhishGuard — Page de test

Chaque lien ci-dessous représente un cas de test pour l'extension PhishGuard.
L'extension doit colorier les liens selon leur niveau de risque détecté.

Prérequis :
1. Backend Flask actif → python api.py (port 5050)
2. Extension chargée dans Chrome → chrome://extensions · mode développeur · "Charger l'extension non empaquetée" → dossier extension/
3. Ouvrir cette page dans Chrome — les liens seront coloriés automatiquement après quelques secondes.

Note : les URLs pointent vers des domaines fictifs ou réservés — ne pas cliquer, le but est uniquement de tester la détection.

SAFE — URLs légitimes (bleu par défaut)

Site institutionnel HTTPS
https://www.google.com
SAFE Aucun signal détecté
GitHub officiel
https://github.com/torvalds/linux
SAFE Domaine connu, HTTPS, rank élevé
Wikipedia
https://fr.wikipedia.org/wiki/Phishing
SAFE TLD standard, pas de signal
Streamlit docs
https://docs.streamlit.io
SAFE Sous-domaine légitime

LOW — Signal faible (bleu clair)

URL longue sans autre signal
https://www.example-legitimate.com/some/very/long/path/…
LOW Longueur URL → score lexical faible
HTTP simple (pas de HTTPS)
http://monsite-perso.fr/blog
LOW Absence HTTPS +15 pts lexical

MEDIUM — Signal modéré (jaune)

Mot-clé "login" + HTTP
http://login.monbanque.fr/verify
MEDIUM login + verify + HTTP
Sous-domaines multiples
https://secure.account.update.service.exemple.com
MEDIUM 4 sous-domaines + mots suspects
TLD .xyz + mot-clé
http://bank-secure.xyz/account/confirm
MEDIUM TLD inhabituel + confirm + bank
URL avec caractères spéciaux
https://ma~banque--securite.com/signin~verify
MEDIUM Chars spéciaux + mots suspects

HIGH — Signal fort (orange)

IP directe dans l'URL
http://192.168.1.254/admin/login
HIGH IP directe +35 pts · login
Caractère @ (redirection masquée)
https://www.google.com@evil-phishing.net/steal
HIGH @ masque le vrai domaine
Typosquatting — gooogle
https://www.gooogle.com/account/signin
HIGH Levenshtein(gooogle, google) = 1
Typosquatting — paypa1
https://paypa1.com/login/secure
HIGH Levenshtein(paypa1, paypal) = 1
Namespace privé — collision AD (.corp)
http://portail.authentification.corp/admin
HIGH .corp fuite namespace + plancher 50
Namespace privé (.internal — réservé ICANN 2024)
https://vpn.entreprise.internal/connect
HIGH .internal officiellement réservé

CRITICAL — Menace confirmée (rouge · notification)

Namespace réservé RFC 2606 — .localhost
http://paypal.localhost/signin/verify
CRITICAL RFC 2606 + typosquatting paypal
Namespace réservé RFC 2606 — .test
https://secure-banking.test/account/update
CRITICAL .test non-résolvable + mots suspects
Infrastructure DNS — .arpa (variante 2025)
http://login.paypal.com.phishing.in-addr.arpa/verify
CRITICAL .arpa jamais légitime + plancher 80
TLD extension de fichier — .zip
https://document-fiscal-2024.zip/telecharger
CRITICAL .zip imite une archive
TLD extension de fichier — .mov
https://video-confidentielle.mov/preview
CRITICAL .mov imite fichier vidéo
Combinaison maximale de signaux
http://192.168.0.1@paypall.test/account/signin/…
CRITICAL IP + @ + .test + typo + mots suspects
